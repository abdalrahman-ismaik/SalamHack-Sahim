# Implementation Plan: Halal Investor Tools Suite (003)

**Branch**: `003-halal-investor-tools` | **Date**: 2026-04-28 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-halal-investor-tools/spec.md`

## Summary

Build 4 P1 Islamic investor tools identified in the competitive gap analysis (constitution Principle X): a **Risk Tolerance Wizard** (8-question weighted questionnaire → Conservative / Moderate / Aggressive profile), a **Dividend Purification Calculator** (inline on the Halal Panel, using existing `interest_income_ratio`), a **Zakat Calculator** (2.5% formula with live nisab from Twelve Data gold price), and **Halal Compliance Change Alerts** (Pro-tier, Firestore-backed, in-app notifications on Dashboard refresh).

All 4 tools are Arabic-first, beginner-safe, and include mandatory Shariah disclaimers and "كيف يُحسب هذا؟" methodology sections. The Risk Wizard and Purification Calculator require no backend changes. The Zakat Calculator requires one new backend endpoint (`GET /api/tools/gold-price`). The Compliance Alerts system uses Firebase Firestore (client-side) for Pro-tier preference storage — no new backend database required.

## Technical Context

**Language/Version**: Python 3.11 (FastAPI backend) · TypeScript 5.x / Next.js 14 App Router (frontend)  
**Primary Dependencies**: FastAPI, Pydantic v2, httpx, NumPy/statsmodels (existing); Next.js 14, Tailwind CSS, Framer Motion, react-hook-form + zod, next-intl, Firebase SDK v9 (Firestore added to existing Auth setup)  
**Storage**: Firebase Firestore (client-side, new — for RiskProfile + ComplianceAlertPreference); no backend database required  
**Testing**: pytest (backend unit tests); Playwright/manual browser (frontend integration); no new test framework introduced  
**Target Platform**: Web (Netlify frontend + Render.com backend); responsive 360px–1280px; RTL Arabic + LTR English  
**Project Type**: SaaS web application — bilingual, multi-tier, Islamic fintech  
**Performance Goals**: Gold price endpoint ≤ 500ms (cached 1hr); Wizard result render ≤ 100ms (client-side only); Zakat calculation ≤ 200ms; Dashboard compliance check ≤ existing halal endpoint latency (~2s per ticker, parallelized)  
**Constraints**: Hackathon deadline May 1 23:59 · No new API keys required · No backend database migration · Guest-accessible for Risk Wizard and Purification Calculator · Server-side tier enforcement via existing JWT tier claim

## User Stories

> Natural language stories grounding each implementation phase.

### US1 — Risk Tolerance Wizard

**As a** new Muslim investor who has never invested before,  
**I want to** answer a short quiz in Arabic about my financial situation and goals,  
**so that** I get a clear, actionable risk profile (محافظ / متوازن / جريء) that tells me what kind of stocks to consider.

*Acceptance*: 6 questions → instant score → colored label badge → 3 concrete recommendations → "كيف يُحسب هذا؟" → disclaimer. Works without sign-in. Profile saved to localStorage for guests, Firestore for logged-in users.

---

### US2 — Dividend Purification Calculator

**As a** Muslim investor who holds a stock flagged as "يحتاج إلى تطهير",  
**I want to** enter my dividend amount and immediately see exactly how much I must donate to charity,  
**so that** my income is purified without having to guess or calculate manually.

*Acceptance*: Embedded in the Halal Panel below the compliance badge. Uses the same `interest_income_ratio` the platform already has. Shows purification amount in the user's chosen currency. Hidden for Halal-status stocks. Real-time calculation (no submit).

---

### US3 — Zakat Calculator

**As a** Muslim investor who wants to fulfil Zakat on my portfolio,  
**I want to** enter my portfolio value and deductible liabilities and see my Zakat obligation in my currency,  
**so that** I know the exact amount to pay without consulting a spreadsheet.

*Acceptance*: Dedicated `/tools/zakat` page. Nisab from live gold price (85g × live price). Shows nisab value and data source. Zakat = 2.5% if net value ≥ nisab, else "أقل من النصاب" message. "Static fallback" badge when API is down. Requires sign-in (Free tier).

---

### US4 — Halal Compliance Change Alerts

**As a** Pro-tier Muslim investor who monitors multiple halal stocks,  
**I want to** be notified on my Dashboard when any of my watched stocks changes compliance status (e.g., from Halal to PurificationRequired),  
**so that** I can re-evaluate my portfolio without manually re-checking each stock.

*Acceptance*: Toggle on each Dashboard stock card. On next Dashboard load, compliance status is fetched and compared to stored value. Notification banner appears at top for any change. Notifications dismiss individually. Free-tier users see "ترقّ إلى Pro" prompt on toggle. Requires Firebase Firestore for preference storage.

### Phase B — Backend: Gold Price Endpoint

*1 new service function + 1 new router + 1 new Pydantic model.*

**Goal**: Provide live nisab basis for the Zakat Calculator.

**Files**:
- `backend/app/models/tools.py` (or append to `stock.py`) — `GoldPriceResponse` Pydantic model with `Literal["TwelveData", "static"]` source field
- `backend/app/services/gold_service.py` — `async get_gold_price() -> dict`:
  - Calls Twelve Data `/quote?symbol=XAUUSD` via existing `httpx` client
  - Converts troy oz price → per gram (÷ 31.1035)
  - Computes AED and SAR equivalents using fixed pegs (3.6725, 3.75)
  - Catches any exception → returns static fallback values (`source="static"`)
- `backend/app/api/tools.py` — FastAPI `APIRouter` with `GET /gold-price`; uses existing `cache_or_fetch("gold_price", get_gold_price, ttl=3600)` pattern
- `backend/app/main.py` — MODIFY: `app.include_router(tools_router, prefix="/api/tools")`

**Acceptance**:
- [ ] `GET /api/tools/gold-price` returns 200 with all 3 currency fields
- [ ] Response is cached for 1 hour (second immediate request returns same value)
- [ ] Endpoint returns `source="static"` with 200 when Twelve Data is unavailable (wrong key or timeout)

### Phase D — Compliance Change Alerts (Pro)

*Depends on Firestore initialization.*

**Goal**: Ship US4 — Pro-tier alerts on Dashboard.

**Files**:
- `frontend/src/lib/firebase.ts` — MODIFY: add `import { getFirestore } from 'firebase/firestore'` and `export const db = getFirestore(app)`
- `frontend/src/hooks/useComplianceAlerts.ts` — reads `users/{uid}/alert_preferences/*` from Firestore; for each enabled ticker, calls existing `GET /api/stock/{ticker}/halal`; diffs `current_status` vs `last_known_status`; returns `ComplianceChangeNotification[]` and saves updated status back to Firestore
- `frontend/src/components/ComplianceAlertToggle.tsx` — toggle component; Free-tier click → opens `UpgradeModal`; Pro-tier click → write to Firestore via `useComplianceAlerts`
- `frontend/src/components/ComplianceNotificationBanner.tsx` — renders notification list; `aria-live="polite"` for accessibility; Framer Motion fade-in animation; dismiss button per notification
- Dashboard page — MODIFY: import and render `<ComplianceNotificationBanner>` and add `<ComplianceAlertToggle>` to each stock card
- `frontend/src/messages/en.json` + `ar.json` — add `alerts.*` keys
- `frontend/src/lib/types.ts` — add `ComplianceAlertPreference`, `ComplianceChangeNotification` types

**Firestore Security Rules** (document in quickstart, implement in Firebase Console):
```
match /users/{uid}/alert_preferences/{ticker} {
  allow read, write: if request.auth != null && request.auth.uid == uid;
}
match /users/{uid}/risk_profile {
  allow read, write: if request.auth != null && request.auth.uid == uid;
}
```

**Acceptance**:
- [ ] Toggle hidden/disabled for Free-tier users; clicking it opens `UpgradeModal`
- [ ] Pro user enables alert for a ticker → preference saved to Firestore
- [ ] On next Dashboard load, if status changed → banner appears at top
- [ ] Banner dismisses individually; dismissed notifications do not re-appear until next status change
- [ ] Notification banner has `aria-live="polite"` for screen reader announcements

1) Problem statement

Many beginners in the Arab world want to invest, but they face 4 problems:

1. Most investing tools are built for advanced users.


2. Financial news is overwhelming and usually English-first.


3. People do not know how to connect news to actual market impact.


4. Muslim users often do not know whether a stock is Sharia-compliant.



2) Solution

Build a platform/app that does 5 things:

1. Collects financial news about companies, sectors, and markets.


2. Uses LLMs to summarize the news in simple Arabic and explain:

what happened

why it matters

who is affected

whether the impact is short-term or long-term



3. Runs quantitative analysis on stocks:

trend

volatility

momentum

support/resistance

valuation/fundamental ratios



4. Scores opportunities for beginners using a simple “traffic light” system:

Green = understandable / lower complexity

Yellow = medium risk / needs caution

Red = high complexity or unclear fundamentals



5. Flags Sharia compliance

compliant

questionable

non-compliant

with explanation

5) Best API/services stack to use

Here is the cleanest practical stack.

1. LLM / reasoning layer

OpenAI Responses API

Use it for:

Arabic summaries

beginner-friendly explanations

structured extraction from news

reasoning over multiple news articles

generating risk explanations and “why this matters” outputs


OpenAI’s Responses API is positioned as the main interface for new applications and supports tool use and function calling, which is useful for a finance copilot flow. 

2. Market data API

Pick one main provider for MVP:

Best choices:

Twelve Data

Alpha Vantage

Finnhub


Why:

Twelve Data has broad global market coverage, technical indicators, fundamentals, analyst data, and market data across stocks, ETFs, forex, crypto, and more. Its docs say it covers over 50 countries and more than 1 million instruments. 

Alpha Vantage offers historical and real-time market data, market news and sentiment, and 50+ technical indicators. 

Finnhub provides real-time market data, company fundamentals, economic data, and alternative data. 


My recommendation:
For a hackathon MVP, use Alpha Vantage or Twelve Data.
If you want the easiest “news + market indicators” combo in one place, Alpha Vantage is attractive. If you want broader instrument coverage and richer fundamentals/indicator endpoints, Twelve Data is strong. 

3. News API

Use two layers:

Primary: NewsAPI
Good for simple news ingestion and it supports a language parameter including Arabic (ar). 

Secondary / stronger global coverage: GDELT DOC 2.0 API
GDELT is powerful because it monitors global news and supports cross-language search through machine-translated coverage; its docs say the translingual system processes 65 languages and translates monitored non-English coverage into English for processing. 

My recommendation:

For fast MVP: NewsAPI

For stronger global/Arabic/news breadth: add GDELT


4. Sharia compliance API/service

Best direct option: Musaffa B2B API

Musaffa publicly documents a B2B API for Sharia-compliant stock screening. 

This is the simplest route if you can get access.

5. Sharia rules fallback engine

If you cannot get a commercial Sharia screening API in time, build your own rules engine using:

company fundamentals from Twelve Data / Alpha Vantage / Finnhub

sector/business descriptions

screening logic based on Islamic finance standards

manual scholar-review disclaimer


AAOIFI is a major Islamic finance standards body and notes that its Shari’ah standards are adopted by a number of central banks and financial authorities either mandatorily or as guidance. 

So your logic can be described as: “AAOIFI-inspired screening rules + fundamentals data + business activity classification.”

For hackathon judging, that is actually impressive because it shows you can still operate even without a commercial halal-data partner.

6. Arabic language support

You have two paths:

Let the LLM do Arabic summarization directly

Add translation/NLP support if needed


Useful services:

Google Cloud Translation for translation pipelines; Google says Cloud Translation supports neural machine translation and API-based translation workflows. 

Azure AI Language if you want extra sentiment/key phrase pipelines and Arabic language support. Microsoft’s language support docs list Arabic support across multiple text features including sentiment-related tasks. 


For MVP, the simplest approach is: LLM for Arabic summaries + optional translation fallback

Submission checklist
1) Cover slide
Add project logo
Add project name
Add one clear tagline
Make sure branding is consistent and readable

Check before submitting

Project name is large and clear
Tagline explains the idea in one line
Logo is high quality
No placeholder text remains

Speaker note
“We are presenting [Project Name], a solution that helps [target users] by [main value].”

2) Problem
Define the problem clearly
Identify who is affected
Explain why the problem matters
Add one or two facts, examples, or pain points

Check before submitting

Problem is specific, not vague
Audience understands who suffers from it
Impact is obvious
Problem matches the solution slide

Speaker note
“The problem is that [user group] struggles with [problem]. This matters because [impact].”

3) Solution
Explain the solution simply
Show how it solves the problem
Highlight what makes it useful or different
Keep the message outcome-focused

Check before submitting

Solution is easy to understand in under 30 seconds
Clear connection to the problem
Main benefit is visible
No unnecessary technical detail

Speaker note
“Our solution is [product/service]. It solves this by [how it works], which helps users [benefit].”

4) Prototype / Demo / MVP
Show what has already been built
Include screenshots, flows, or demo points
Highlight key features
Show what is functional today

Check before submitting

Prototype is real and visible
Screenshots are readable
Main user flow is clear
MVP scope is realistic

Speaker note
“We have built [prototype/demo/MVP]. Users can already [main actions], which shows the concept is practical.”

5) Tech Stack
List the main technologies used
Group by frontend, backend, database, AI, cloud, APIs, etc.
Mention only tools that matter
Keep the slide clean

Check before submitting

Stack is organized
Terms are accurate
No overcrowding
Technical choices support the solution

Speaker note
“Our product is built using [main tools]. We chose this stack because it supports [speed, scalability, AI capability, reliability, etc.].”

6) Business Model
Define target users or customers
Explain how revenue will be generated
Show market demand
Add supporting numbers or market statistics

Check before submitting

Customer segment is clearly defined
Revenue model is realistic
Market opportunity is supported by data
Slide answers why this can become sustainable

Speaker note
“Our target customers are [segment]. We plan to generate revenue through [model]. The opportunity is strong because [market fact/statistic].”

7) Investment pitch
State whether funding is needed
Specify the amount
Explain how the money will be used
Describe ROI potential and growth plan

Check before submitting

Funding ask is specific
Use of funds is practical
Growth plan is believable
Investor value is clear

Speaker note
“We are seeking [amount] to invest in [product, growth, hiring, partnerships, etc.]. This will help us reach [milestone] and create value through [growth path].”

8) Team
Add all team members
Include name
Include country
Include specialization or field of study
Include role in the project

Check before submitting

Roles are clear
Team strengths match the project
Names and details are spelled correctly
Everyone’s contribution is visible

Speaker note
“Our team combines strengths in [areas]. Each member contributes directly to [product, design, business, tech, research, etc.].”

Final review checklist
All placeholder text removed
Slide titles match content
Fonts and spacing are consistent
Arabic and English text are formatted cleanly
Each slide can be explained in under 30 to 45 seconds
Problem, solution, and business model all align
Demo is ready if you plan to present it live
Numbers and claims are accurate
Team roles are clear
Deck tells one connected story
Suggested speaking flow
Introduce the project
Show the problem
Present the solution
Prove it with the prototype
Show how it is built
Explain how it can grow as a business
Present the investment case
End with the team
